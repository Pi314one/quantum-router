1. 简介

gjq-client 是一款面向国基量子云平台的 Python 客户端工具，致力于为用户提供高效、便捷的量子线路构建、转译、任务提交与结果获取能力。无论是量子计算初学者，还是从事科研与工程开发的专业人员，都可以借助 gjq-client 快速接入云端量子资源，开展量子算法的设计、验证与实验。
本教程将系统介绍 gjq-client 的安装与环境配置、核心功能的使用方法，以及若干进阶实践技巧，帮助用户全面掌握该工具并提升量子开发效率。除量子真机外，平台当前还接入了两个云端量子模拟器 SAS-CPU 与 FAS-CPU，便于用户在本地计算资源不足时直接调用云端算力完成量子线路仿真。

2. 安装gjq-client

2.1 通过 pip 安装 gjq-client

gjq-client支持通过 PyPI（pip）直接安装，也可从源码构建安装。下面给出在 Windows + PowerShell 环境下的推荐步骤及常见问题说明，其他环境类似，这里不再赘述。

  • 推荐方式（稳定、便捷）：
# 在 PowerShell 中运行：
python -m pip install --upgrade pip
python -m pip install gjq-client

说明：

  • PyPI 上的发行包名为 gjq-client，安装后在 Python 中通过 import gjq_client 使用。
  • 建议在虚拟环境中安装，以避免与系统环境产生依赖冲突。
  • 若使用虚拟环境（推荐），请先创建并激活 venv：
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install gjq-client

问题排查：

  • 如果提示找不到版本或无法安装，请检查 Python 版本（本项目要求 Python >= 3.10）。
  • 若企业网络或代理导致 pip 失败，可通过 --proxy 参数或离线 wheel 安装。

2.2 通过源码安装gjq-client

当需要修改源码或在开发过程中调试时，推荐使用源码安装（可编辑安装）。在仓库根目录（包含 pyproject.toml）下运行：

# 克隆仓库
git clone https://github.com/guoji-quantum/gjq_client.git
cd gjq_client

# 可编辑安装，便于调试源码
python -m pip install -e .

注意：若本地修改了 src/gjq_client 下的代码，使用 pip install -e . 可以使 Python 环境直接使用工作区中的源文件，便于单步调试与快速迭代。

2.3 账户配置

gjq-client 通过 API Key 与国基量子云平台进行认证，用户账号信息在 GJQRuntimeService 中配置和使用。流程如下：

  1. 获取 API Key：访问 国基量子云平台 ，注册并登录后可以看到如下的页面，复制API key
  2. 创建服务实例
from gjq_client import GJQRuntimeService

service = GJQRuntimeService(
    api_key="your_api_key",
)

默认配置文件位置：

  • Windows: %USERPROFILE%/.gjq_client/gjq_client_account.json
  • Linux: ~/.gjq_client/gjq_client_account.json
  • macOS： ~/.gjq_client/gjq_client_account.json

该文件保存了 channel、api_key、base_url 等信息。若在构造函数提供了 api_key，SDK 会同步更新配置文件中的信息，若未提供 api_key，SDK 会从默认配置文件中读取配置。

安全建议：

  • 切勿将包含 api_key 的配置文件上传到公共代码仓库。
  • 建议为 CLI 或自动化环境创建仅限于所需权限的 API Key，并按需定期轮换。

3.线路操作

gjq-client 与 Qiskit 深度集成（qiskit>=2.0），用户在提交任务前需使用 Qiskit.QuantumCircuit() 构建量子线路，并通过 SDK 提供的转译工具确保量子线路符合云端后端约束。

  • 当用户直接提交源代码到云平台时 (optimization_level=None)，线路使用的量子门需限制在以下集合内：
[h, cz, cx, ccx, swap, cswap, x, y, z, s, sdg, t, tdg, sx, rx, ry, rz, cp, rxx, ryy, rzz]

说明：用户在转译量子线路时，会有 optimization_level 选项，默认情况下 optimization_level=2。当 optimization_level=None 时，不会进行自动转译，会直接提交源量子线路；若使用 0~3 的任一数值，系统会对线路进行相应的转译与优化，届时可使用的门集合不再受上述限制。

  • 如何在 Qiskit 中使用这些门：
from qiskit import QuantumCircuit
qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.ccx(0,1,2)
qc.rxx(0.5, 0, 1)

注意事项与建议：

  • 如果使用了后端不支持的自定义门或复杂的控制流，请在提交前使用后端对象的 check_faulty、properties 与 configuration() 方法校验线路是否符合设备约束（例如量子比特数、错误率或故障量子比特）。
    示例：检查线路是否超出后端限制
service = GJQRuntimeService(api_key="your_api_key")
backend = service.backend("backend_name")
qc = QuantumCircuit(5)
# ... 构造量子线路
backend.check_faulty(qc)  # 如果包含故障 qubit 或边，会抛出 ValueError

4. 转译

在用户完成量子线路构建之后，通常不能直接提交至量子真机执行，而需要先进行转译（transpile）以适配目标量子硬件的实际约束条件。这一步是量子程序从“抽象线路”走向“可执行线路”的关键环节，其核心目的是将硬件无关的逻辑线路转换为满足特定量子设备拓扑结构、原生门集以及误差特性的低层实现。进行转译时，需要先获取目标 backend对象，然后通过 generate_preset_pass_manager 函数构造 PassManager 对象，并调用其 run 方法以获得转译后的线路。

4.1 获取backend

获取backend有两种方法

  1. 自动选择空闲量子真机：使用 GJQRuntimeService().least_busy() 可以获取当前排队任务最少的量子真机。这种方式适合在不确定具体设备、希望快速提交任务时使用。示例代码：
from gjq_client import GJQRuntimeService
service = GJQRuntimeService(api_key='')
backend = service.least_busy()
  1. 直接指定后端名称：如果需要在特定设备上运行线路，可通过后端名称直接获取对应 backend。这里的后端既可以是量子真机，也可以是云端量子模拟器，适用于对设备特性有特殊要求的场景。示例代码：
from gjq_client import GJQRuntimeService
service = GJQRuntimeService(api_key='')
backend = service.backend('Baihua')  # 指定量子真机名称

如果希望使用云端模拟器，也可以直接通过名称获取。其中：

  • SAS-CPU：单振幅模拟器；
  • FAS-CPU：全振幅模拟器。

示例代码：

from gjq_client import GJQRuntimeService

service = GJQRuntimeService(api_key='')
sas_backend = service.backend('SAS-CPU')  # 单振幅模拟器
fas_backend = service.backend('FAS-CPU')  # 全振幅模拟器

说明：如果用户不知道可用后端名称，可以通过 GJQRuntimeService().list_backends() 方法获取当前账号下所有可用后端列表（包括量子真机和云端模拟器）。示例代码：

from gjq_client import GJQRuntimeService
service = GJQRuntimeService()
backend_list = service.list_backends()   # 获取所有可用后端列表

在返回结果中，用户可根据后端名称选择真机或模拟器。例如当任务只需使用云端模拟能力时，可优先选择 SAS-CPU 或 FAS-CPU。

4.2 转译优化

转译优化时，先通过 generate_preset_pass_manager 函数构造 PassManager 对象，并调用其 run 方法以获得转译后的线路。示例代码：

from gjq_client import GJQRuntimeService,generate_preset_pass_manager
from qiskit import QuantumCircuit

qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.ccx(0,1,2)
qc.rxx(0.5, 0, 1)

service = GJQRuntimeService(api_key='')
backend = service.backend('Baihua')  # 指定量子真机名称
pm = generate_preset_pass_manager(optimization_level=2, backend=backend)
isa_circuit=pm.run(qc)

优化等级（optimization_level）说明：

  • None：即不对线路进行任何自动转译或优化（直接提交源码/原始门序列）。
  • 0：最低程度的处理，仅做必要的格式调整与合规性转换（例如把控制流保持原样，但不做深度优化）。
  • 1：轻量优化，适用于对时延与门数有中等敏感性的场景。
  • 2（默认）：较强优化，通常是最常用的等级，在保持正确性的同时尽量减少量子门深度/数目。
  • 3：最高等级，使用更激进的优化与合并策略，可能会改变表达形式以换取最小的深度与门数。

注意与实践建议：

  • 如果想完全控制转译流程（例如自定义 routing、layout 或 translation 插件），可以构造 Target 并传入 generate_preset_pass_manager(..., target=your_target)。
  • 如果设置 optimization_level=None，SDK 会返回不包含默认转译流程的 PassManager，适用于用户希望以原生门序列或自行控制转译流程直接提交线路的场景。
    性能提示：
  • 转译可能会消耗较多 CPU（尤其是 level 2/3），建议在客户端本地进行转译时使用可用的并行资源或在构建流水线中缓存转译结果以便重复利用。

5. 任务提交

完成量子线路构建与转译后，即可将线路提交至量子后端执行。目标后端既可以是量子真机，也可以是云端量子模拟器（如 SAS-CPU、FAS-CPU）。本 SDK 提供 Sampler 与 Estimator 两种任务提交方式，分别用于获取采样分布结果与期望值结果，提交任务后，云平台会返回该任务的唯一ID instanceId，后续查询任务状态和结果都是通过该任务ID来查询。以下给出完整使用流程。

5.1 Sampler 方式提交任务

Sampler 用于执行量子线路并返回测量计数（counts），适用于算法验证、概率分布分析等场景。

  1. 构造 PassManager 并完成转译
pm = generate_preset_pass_manager(
    backend=backend,
    optimization_level=None
)

参数说明：

  • backend：目标量子后端
  • optimization_level：
    • None：不进行转译，直接提交原始线路
    • 0~3：不同级别的硬件适配与优化
  1. 运行转译
isa_circuit = pm.run(qc)
  1. 创建 Sampler 并提交任务
# 注：当后端选择云端单振幅模拟器 `SAS-CPU` 时，需要指定要查询的量子态振幅索引 `amplitude_index`。该索引表示计算基态在 statevector 中的十进制位置，例如 `amplitude_index=[4]` 表示获取基态 |100⟩ 的振幅。若后端为 `FAS-CPU` 或量子真机，则无需传入该参数。
sampler = Sampler(backend=backend)  
#job = sampler.run(isa_circuit, shots=1024), amplitude_index=[4]) #`SAS-CPU` 需要该参数
job = sampler.run(isa_circuit, shots=1024)

参数说明：

  • isa_circuit：转译后的线路
  • shots：采样次数
  • amplitude_index：仅在 SAS-CPU 后端下使用，用于指定要返回的目标振幅索引
  1. 获取运行结果：任务提交后会返回一个 RuntimeJob 对象。调用其 result() 方法时，客户端会进入阻塞状态，持续轮询云端任务执行进度，并在任务完成后返回最终结果。
print(job.result())
  • 返回示例
{
    "instanceId": "3a7cd679-592e-4b7e-b0d4-af2d2e51b9c8",
    "submit_date": "2026-04-09T15:40:48Z",
    "finish_date": "2026-04-09T15:40:57Z",
    "counts": {"0100": 123, "0110": 40, "0000": 252, "1000": 242, "0010": 109, "1010": 84, "1001": 2, "1110": 38, "1100": 127, "1101": 2, "0001": 3, "0101": 1, "0011": 1}
}
  1. 完整示例
from gjq_client import GJQRuntimeService,generate_preset_pass_manager,Sampler
from qiskit import QuantumCircuit

qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0,1)
qc.ccx(0,1,2)
qc.rxx(0.5, 0, 1)

service = GJQRuntimeService(api_key='')
backend = service.backend('SAS-CPU')  # 以云端单振幅模拟器为例；若使用 FAS-CPU，可直接替换后端名称
pm = generate_preset_pass_manager(optimization_level=2, backend=backend)
isa_circuit=pm.run(qc)

sampler = Sampler(backend=backend)
job = sampler.run(isa_circuit, shots=1024, amplitude_index=[4])

print(job.result())

5.2 Estimator方式提交任务

Estimator 用于计算量子算符的期望值（Expectation Value），适用于能量估计、量子优化等场景。

  1. 构造量子算符
from qiskit.quantum_info import SparsePauliOp

M1 = SparsePauliOp.from_list([("XX", 1)])  # 定义测量算符
  1. 对量子线路添加基旋转和测量：在使用 Estimator 计算量子算符的期望值时，量子算符通常包含 X、Y、Z 等不同基的 Pauli 算符。为了在硬件或后端上测量期望值，需要将测量转换到 Z 基，因为大多数量子硬件默认只能直接进行 Z 基测量。示例代码：
qc = QuantumCircuit(2, 2)
qc.h(0)
qc.cx(0, 1)

for i, p in enumerate(M1.paulis[0].to_label()):
    if p == 'X':
        qc.h(i)      # X -> Z 基旋转
    elif p == 'Y':
        qc.sdg(i)    # Y -> X
        qc.h(i)      # X -> Z
qc.measure_all()
  1. 构造 Pass Manager 并完成转译
from gjq_client import generate_preset_pass_manager

pm = generate_preset_pass_manager(
    backend=backend,
    optimization_level=0  # 0~3 为优化等级，None 表示不转译直接提交源线路
)
  1. 运行转译
isa_circuit = pm.run(qc)
  1. 创建 Estimator 并提交任务
from gjq_client import Estimator
# 注：在云端单振幅模拟器 `SAS-CPU` 中使用 Estimator 方式并不具有物理意义。为保持接口兼容性，任务仍需按 Estimator 规范提交（包括 obs 等参数），但后端不会执行真实的期望值计算。其返回的期望值（evs）仅为兼容性占位结果：当对应量子态振幅概率为 0 时返回 0，当概率非零（即 counts > 0）时返回 1，不代表实际物理期望值。
# 对于 `FAS-CPU` 或量子真机，则按正常 Estimator 语义返回期望值结果。
estimator = Estimator(backend=backend)
job = estimator.run(isa_circuit, obs=M1)

参数说明：

  • isa_circuit：转译后的量子线路
  • obs：要测量的量子算符
  1. 获取运行结果：任务提交后会返回 RuntimeJob 对象，调用 result() 会阻塞等待任务完成
print(job.result())  # 返回期望值
  • 返回示例：
{
    "instanceId": "d5563855-8ecd-4444-9f7f-dd2a6143d286",
    "submit_date": "2026-04-09T15:40:48Z",
    "finish_date": "2026-04-09T15:40:57Z",
    "evs": 0.6652
}
  1. 完整示例
from gjq_client import GJQRuntimeService, Estimator, generate_preset_pass_manager
from qiskit import QuantumCircuit
from qiskit.quantum_info import SparsePauliOp

# 构造量子线路
bell = QuantumCircuit(2)
bell.h(0)
bell.cx(0, 1)

# 构造算符并添加基旋转
M1 = SparsePauliOp.from_list([("XX", 1)])
for i, p in enumerate(M1.paulis[0].to_label()):  # 本示例仅演示单个 Pauli 项的期望值测量
    if p == 'X':
        bell.h(i)
    elif p == 'Y':
        bell.sdg(i)
        bell.h(i)
bell.measure_all()  #Estimator 需要线路包含显式测量位

# 获取后端并转译
service = GJQRuntimeService(api_key='')
backend = service.backend('Baihua')
pm = generate_preset_pass_manager(backend=backend, optimization_level=0)
isa_circuit = pm.run(bell)

# 提交 Estimator 任务
estimator = Estimator(backend=backend)
job = estimator.run(isa_circuit, obs=M1)

# 获取结果
print(job.result())

5.3 查询与获取结果

在提交量子任务后，gjq-client 提供了多种方法查看任务状态和获取结果。以下示例使用 GJQRuntimeService 类。

  1. 获取正在运行的任务列表：调用 list_tasks() 可以获取当前用户正在运行的任务列表，包括任务 ID、提交时间和状态等信息
from gjq_client import GJQRuntimeService

service = GJQRuntimeService(api_key='your_api_key')
tasks = service.list_tasks()
print(tasks)
  • 示例返回：
[{'instanceId': 'babe3d65-c2d0-4390-b178-dc3829d684e4', 'task_state': 'running'}, {'instanceId': '80455522-15bc-4a8b-992e-99f8673d94e2', 'task_state': 'running'}]
  1. 查询单个任务状态：通过 task_status(instanceId) 可以获取指定任务的当前状态
instanceId = "3c512803-6420-4150-95fd-d150562aec93"
status = service.task_status(instanceId)
print(status)
  • 示例返回：
{'task_id': '3c512803-6420-4150-95fd-d150562aec93', 'status': 'RUNNING'}
  1. 获取任务结果：任务完成后，调用 task_result(instanceId) 获取具体运行结果
instanceId = "80455522-15bc-4a8b-992e-99f8673d94e2"
M1 = SparsePauliOp.from_list([("XX", 1)])

##Estimator 任务，对于 Estimator 任务，需要在获取结果时再次提供 observable，用于计算期望值
result = service.task_result(task_id,obs=M1) 
#Sampler 任务
#result = service.task_result(task_id) 

print(result)
  • 示例返回：
{
  'instanceId': '80455522-15bc-4a8b-992e-99f8673d94e2',
  'submit_date': '2026-04-09T15:40:48Z',

6. 常见问题与调试技巧

在使用 gjq-client 时，用户可能会遇到认证、网络、转译或任务提交等问题。以下整理了一些常见问题及对应调试方法：

6.1 认证失败

  • 问题表现：提交任务或获取结果时返回 401/403 错误，或报 AuthenticationError。
  • 排查与解决：
    1. 检查 api_key 是否正确，确保无多余空格或误输入字符。
    2. 可使用 ClientParameters.authenticate() 获取更详细的认证信息：
from gjq_client.client.client_parameters import ClientParameters  
cp = ClientParameters(api_key="your_api_key")  
cp.authenticate()

6.2 提交任务失败

  • 问题表现：调用 Sampler.run() 或 Estimator.run() 时返回错误或异常。
  • 调试方法：
    1. 确认线路已正确构建并满足后端约束：
    • 线路中的量子门在硬件允许的门集合内。
    • 线路没有超过后端的量子比特数量或拓扑限制。
    1. 对于 Estimator 任务：
    • 确认 isa_circuit 已正确转译（使用 generate_preset_pass_manager）。
    • 量子算符 obs 对应的线路基旋转已正确添加。
    • 线路中必须包含显式测量位（measure_all() 或自定义测量寄存器）。

6.3 其他建议

  • 建议在开发过程中使用 可编辑安装模式(pip install -e .) 以便快速迭代调试。
  • 可通过 GJQRuntimeService.list_tasks()、GJQRuntimeService.task_status()、GJQRuntimeService.task_result() 实时监控任务执行情况。
  • 对于频繁使用的量子线路，可考虑本地缓存转译结果，提高开发效率。
